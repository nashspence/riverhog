# Riverhog Minimal ADR Set

This set rewrites Riverhog's explicit and implied architecture decisions as firm, concise ADRs.

Each ADR has two required sections:

- `Decision`: the firm, unequivocal, low-precision what.
- `Reason`: the actual why.

`Context` is optional and should appear only when the decision or reason would otherwise be unclear to an average PM/dev reading this repo.
